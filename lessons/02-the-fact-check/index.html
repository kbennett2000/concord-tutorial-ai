Ollama isn't running. The 30-second check: open http://localhost:11434 in a new tab — you want "Ollama is running." If not: on Windows and Mac launch the Ollama app; on Linux it's a background service — sudo systemctl start ollama. Then reload. (SETUP has the full path.)
Concord isn't answering. Course 1's ritual, unchanged: open http://localhost:8000/healthz — a short line of data means it's on. If nothing loads, open Docker and start Concord, then reload. (SETUP shows the way.)
